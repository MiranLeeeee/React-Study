## Week3

### 리덕스(Redux)
* 부모 자식간의 상위 데이터 관리가 아닌 전역 데이터 관리가 필요
* 상태관리를 도와주는 라이브러리
* 설치: `yarn add redux react-redux`
* 용어
  1. state: 리덕스에서 저장하고 있는 상태 값, 데이터 -> 딕셔너리로 key,value형태
  2. action: 상태 값이 변화가 필요할 때 발상 시킴
  3. actioncreator: 액션을 생성하기 위해 사용
  4. reducer: 리덕스에 저장된 데이터를 변화시켜주는 메소드
  5. store: 리듀서, 프로젝트, 내장함수 등을 포함
  6. dispatch: 액션을 발생 시키는 역할로 store 내장함수 중 하나
* 액션생성함수를 호출하여 액션을 만들고 리듀서가 현제 데이터값과 액션을 받아 새로운 데이터를 만들어서 반환
* 특징
  1. store는 1개만 씀, 한 프로젝트 당 단일 스토어
  2. store의 state는 오직 action으로만 변경가능 -> action을 통해 state가 변경되는 것을 전역적으로 모든 컴포넌트에게 알려줌
  3. 리듀서는 순수한 함수이므로 어떤 요청이 와도 같은 동작을 행함
* 리덕스를 통해 리액트 상태관리
  1. 컴포넌트는 스토어의 state를 구독 중
  2. 컴포넌트에서 액션이 취해지면 액션을 호출
  3. 리듀서는 디스패치하여 액션과 함께 이전 상태값을 변화시킴
  4. store에 변경된 새 상태값을 저장
  5. 컴포넌트는 새 상태값이 있는 store을 구독
